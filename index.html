The Geospatial data and services portal facilitates the coordinated development, maintenance, and sharing of geospatial data and web map services by Norwegian Polar Instiute.

To meet the ever increasing number of tile-based queries against forum Administration WMS services, we now offer a selection of our basic mapping services that cache services. The services can be used in clients with support for WMS-C as OpenLayers, Google Maps or Microsoft Virtual Earth The purpose of the cache is to achieve fast response times of map requests. A cache service is not as flexible as for example, a WMS service. For example, when a WMS service could select one or more layers while in a cache must select all or nothing. Authority uses free software component GeoWebCache to caching services.

1. Cached Map Services

A cached map service is a regular map service that has been enhanced to serve maps very quickly using a cache of static images. The map cache is a directory that contains image tiles of a map extent at specific scale levels. Returning a tile from the cache takes the server much less time than drawing the map image on demand. Map caching is a very effective way to make your ArcGIS Server maps run faster. When you create a map cache, the server draws the entire map at several different scales, and stores copies of the map images. The server can then distribute these images whenever someone asks for a map. It's much quicker for ArcGIS Server to hand out a cached image than to draw the map each time someone requests it. Another benefit of caching is that the amount of detail in the image doesn't noticeably affect how quickly the server can distribute the copy. Cache services are based on a cache with touches of thousands detailed maps images or tiles at 256x256 pixels at predefined scales (zoom levels), in the same way as Google Maps or Esri Maps is built up. Therefore, a cache service does not provide map images in freely chosen scale, but only in the predefined scales defined in the service.

2. Thematic Map Services

WMS is a service where a server request to send map section in the form of raster files back to the user, either directly in the browser, a web application or a desktop application. This type of service requires some expertise to take in use, and it's the software that makes calls to map server.

services based on WMS specification is flexible and provides the user with many options, such as:
  • Choosing which coordinate chart will be produced in
  • turn on or off layers in the service, for example, you can select only the roads and water to be drawn and everything else omitted
  • Freely choose scale
  • Select the background color, or whether this should be transparent
  • Which format to be produced
  • Retrieving Legend of the various map layers
  • Obtaining basic information about objects on the chart, such as buildings, roads and properties

Limit of WMS calls

Currently there is no limit for WMS calls to users ip address but there is timeout for client using a service and client waiting to get a service.
